短链服务

功能​

  1. 用户可为长链新建一个短链
    • 用户短时间为相同长链建短链，返回相同短链
    • 短链支持设置过期时间
    • 短链生成没有规律，不可预测
  2. 用户访问短链即可跳转到长链
    • 302跳转，每次跳转都要经过server，不走client本地缓存，方便server统计等功能，可以配置成301，302可切换，降低server压力
  3. 高性能，高可用
    • 服务可多副本部署，使用cache加速查询

架构图​

db​

db的选取应考虑数据量，延迟，并发量，高可用等方面选取，然后测试是否符合要求

目前使用mysql。分片，多副本部署可保证高性能高可用，或用其他高性能db

CREATE TABLE short_urls(
   short_key VARCHAR(8) NOT NULL,
   original_url VARCHAR(1024) NOT NULL,

   expire_time DATETIME DEFAULT NULL,

   create_time DATETIME NOT NULL,
   update_time DATETIME NOT NULL,
   PRIMARY KEY (short_key)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='key与长链映射表';

cache​

  1. 缓存最近经常访问的短链

  2. 缓存最近新增的长链，为了支持重复提交一个长链，都返回同一个短链
